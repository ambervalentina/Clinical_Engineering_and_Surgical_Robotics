Confusion matrix: This is not a metric as such but rather the base for multiple metrics to evaluate classification algorithms. Confusion Matrices are simply mapping matrices that compare the outcome of your algorithm and the true label of a specific target. From them, we can obtain 4 basic values: True Positives (TP), True Negatives (TN), False Positives (FP) and False Negatives (FN). The following Metrics are derived from the confusion matrix.

Accuracy (Acc): One of the most used metrics to evaluate a classification algorithm. It is calculated according to the following equation:

The higher this value, the better your algorithm is when it comes to classifying samples. An accuracy of 100% means that your algorithm is perfect when separating the classes. Be careful though: if the data is not balanced (in terms of number of samples per class), the accuracy could be high at the training stage, but the model may not classify unseen (test) data correctly.

Precision: This is a widely used metric to demonstrate how well the classification of the positive class (Cancer in this case) is performing. It indicates how many samples are correctly classified as positive.

Sensitivity/Recall: This is similar to Precision. It is very useful when the target is a rare event (also called imbalanced). It gives us the percentage of samples classified by the algorithm as positive among all the real positive samples.

Specificity: This variable provides the proportion of negatives that our algorithm is identifying correctly. This value is required to calculate the AUC (see below).

F-score: This metric combines Precision and Sensitivity to provide an overarching assessment of the algorithm’s accuracy.

A high F-score means that our model is able to better separate both classes (which is the ultimate goal of this type of classification algorithm).

AUC: This variable represents the area under the ROC curve (and is not derived from the confusion matrix). The ROC curve plots the value of Sensitivity against 1-Specificity for any threshold available in the classification algorithm (and can also be analysed in the Statistics and Machine Learning toolbox. AUC is then the area under the ROC curve. The higher the AUC, the better your model is separating the classes at any threshold. A value of 0 means that your model is worse than a random classification model and a value of 1 represents perfect classification (i.e. correct classification of all samples at all thresholds).

This phenomenon often arises due to overfitting, where the model captures spurious patterns that won't recur in the future (noises and outliners), leading to less accurate predictions.[1]^{[1]}[1]﻿ To improve classification accuracy for the test dataset, we can do some noise reduction or outlier removal.

For example, we can reduce outliners using techniques like the interquartile range (IQR). Here’s the example code to impose IQR on Healthy and Cancer datasets to remove outliers.

% IQR for Cancer and Healthy 
Cancer_IQR = iqr(Cancer');
Healthy_IQR = iqr(Healthy');

% lower and upper quartiles for Cancer and Healthy datasets
Ql_Cancer = quantile(Cancer', 0.25);
Qu_Cancer = quantile(Cancer', 0.75);
Ql_Healthy = quantile(Healthy', 0.25);
Qu_Healthy = quantile(Healthy', 0.75);

% define outlier thresholds
lower_bound_Cancer = Ql_Cancer - 1.5 * Cancer_IQR;
upper_bound_Cancer = Qu_Cancer + 1.5 * Cancer_IQR;
lower_bound_Healthy = Ql_Healthy - 1.5 * Healthy_IQR;
upper_bound_Healthy = Qu_Healthy + 1.5 * Healthy_IQR;

% Filter out outliers
Cancer_filtered = Cancer(:, all(Cancer > lower_bound_Cancer & Cancer < upper_bound_Cancer));
Healthy_filtered = Healthy(:, all(Healthy > lower_bound_Healthy & Healthy < upper_bound_Healthy));
